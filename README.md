# thesis-to-defense-script

把博士或硕士学位论文打磨成正式答辩讲稿的 Codex Skill。它适合在论文已经定稿或基本定稿后，将论文内容压缩、重组并改写成 30-45 分钟答辩用的逐页讲稿。

这个 Skill 的重点不是简单“论文转口语”，而是帮助讲稿形成清楚的递进叙事：从领域背景出发，收敛到具体问题；说明前人研究基础和瓶颈；自然提出科学问题、假说和技术路线；再把结果章节改写成“做了 X → 发现 Y → 因此想到 Z → 去做下一步”的环环相扣结构；最后沿着前面的故事线归纳结论、创新点、局限与展望。

它的核心优势是避免把实验讲成平铺的流水账。很多论文结果章节会写成“我做了实验一、实验二、实验三”，但正式答辩更需要讲出研究者的思考路径：我为什么要做这件事，做完之后得到了什么结果，这个结果说明了什么，又如何自然引出了下一件事。这样讲出来的答辩不是实验清单，而是一条能让评委跟着走的研究故事线。

## 适用场景

- 博士、硕士学位论文答辩讲稿
- 答辩 PPT 逐页讲稿
- 30-45 分钟正式答辩报告
- 把论文绪论压缩成答辩背景页
- 把结果章节改写成递进式口播
- 修改、润色、压缩或扩展已有讲稿
- 整理创新点、结论、局限与展望
- 准备读音约定、备答页和常见追问

## 核心能力

- 从论文绪论中提炼“领域大背景 → 具体问题 → 前人基础 → 研究瓶颈 → 可行方向 → 科学问题 → 假说 → 目标 → 技术路线”的递进骨架。
- 把结果章节从“实验 1、实验 2、实验 3”的平铺结构，改写成“承上 + 方法/结果 + 启下”的逐页讲稿。
- 为每个实验补上“为什么要做”：上一页留下了什么问题，本页实验如何回应这个问题。
- 为每个结果补上“说明了什么”：避免只读图或罗列数据，而是解释结果对假说、问题和下一步实验的意义。
- 为章节之间补上“怎么引出下一步”：让实验之间形成因果推进，而不是互不相干的并列模块。
- 保留正式答辩需要的专业表达，例如“科学问题”“关键瓶颈”“研究缺口”“技术路线”“验证范围与材料层面”“机制层面”“证据链条”“逻辑闭环”等。
- 替换过度口语化的行业黑话，例如把“立住了”改为“得到验证/成立”，把“卡点”改为“瓶颈/制约因素”，把“故事线”改为“研究脉络”。
- 将散落在正文里的谨慎限定集中整理到“局限与展望”页，避免打断主体叙事。

## 使用示例

```text
请使用 thesis-to-defense-script skill，根据我的论文和 PPT 大纲，写一版 40 分钟正式答辩逐页讲稿。
```

```text
请帮我修改这版答辩讲稿，让结果章节不要平铺，而是每一步都从上一步自然引出。
```

```text
请把我的局限与展望页改得更稳妥，不要像是在承认实验设计缺陷，而是写成可继续推进的方向。
```

```text
请检查讲稿里是否有“立住了、卡点、对得上、跑下来、故事线”这类不适合正式答辩的说法，并替换成正式表达。
```

## 输入建议

为了获得更好的效果，建议提供：

- 论文绪论、摘要、结论或完整论文
- PPT 页码、标题和每页要点
- 目标答辩时长，例如 30 分钟、40 分钟或 45 分钟
- 已有讲稿草稿
- 希望重点强调或避免展开的内容
- 答辩学科背景和听众情况
